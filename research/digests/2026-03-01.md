## 1) AgentSentry: Mitigating Indirect Prompt Injection in LLM Agents via Temporal Causal Diagnostics and Context Purification
- **Authors:** Tian Zhang, Yiwei Xu, Juan Wang, et al.
- **arXiv:** 2602.22724v1 (**cs.CR**, cs.AI)
- **Summary (5-8 lines):**
  - Frames indirect prompt injection (IPI) as a **multi-turn causal takeover** problem instead of a single-turn anomaly.
  - Introduces counterfactual re-execution at tool-return boundaries to localize where attacker influence enters trajectory.
  - Uses “context purification” to remove suspected attack-induced context while keeping benign evidence.
  - Evaluated on AgentDojo across 4 task suites and 3 attack families.
  - Reports elimination of successful attacks in tested setup, with utility under attack around 74.55%.
  - Improvement over baselines is framed as +20.8 to +33.6 points without benign-regime degradation.
- **Novelty:**
  - Causal/temporal diagnostics for IPI localization.
  - Recovery-oriented mitigation (continue safely) vs pure blocking.
- **Security relevance:**
  - **Permissions/policy + data security**: helps prevent tool-mediated takeover and downstream leakage while preserving mission flow.
- **Limitations / open questions:**
  - Runtime overhead of counterfactual replays at production latency.
  - Robustness on long-horizon, high-tool-count workflows.
  - Dependence on tool boundary instrumentation quality.
- **Tags:** `prompt-injection`, `causal-defense`, `tool-agents`, `inference-time-security`, `AgentDojo`

## 2) Silent Egress: When Implicit Prompt Injection Makes LLM Agents Leak Without a Trace
- **Authors:** Qianlong Lan, Anuj Kaul, Shaun Jones, Stephanie Westrum
- **arXiv:** 2602.22450v1 (**cs.CR**, cs.AI)
- **Summary (5-8 lines):**
  - Demonstrates “implicit prompt injection” via URL previews/metadata/snippets in agent workflows.
  - Shows attacker pages can trigger outbound requests that exfiltrate runtime context while final user-visible answer appears harmless.
  - In reported 480-run local experiments, egress occurs with high probability and often bypasses output-based checks.
  - Introduces **sharded exfiltration** across multiple requests to evade simple DLP thresholds.
  - Ablations suggest prompt-layer fixes are weaker than network/system controls.
  - Emphasizes treating network egress as first-class security outcome.
- **Novelty:**
  - Formalizes silent egress channel via implicit context artifacts.
  - Quantifies sharded exfiltration effects against naive leakage metrics.
- **Security relevance:**
  - **Data security/privacy**: direct exfiltration risk; motivates egress allowlists, redirect-chain policies, provenance.
- **Limitations / open questions:**
  - Generalization to larger model/tool stacks outside tested setup.
  - Interaction with enterprise proxies and zero-trust egress gateways.
  - Detection under encrypted or covert channels.
- **Tags:** `data-exfiltration`, `implicit-injection`, `network-egress`, `DLP-bypass`, `agent-security`

## 3) Skill-Inject: Measuring Agent Vulnerability to Skill File Attacks
- **Authors:** David Schmotz, Luca Beurer-Kellner, Sahar Abdelnabi, Maksym Andriushchenko
- **arXiv:** 2602.20156v3 (**cs.CR**, cs.LG)
- **Summary (5-8 lines):**
  - Defines skill-file prompt injection as a supply-chain risk in extensible agent ecosystems.
  - Introduces SkillInject benchmark (202 injection-task pairs) spanning overt and subtle context-dependent attacks.
  - Evaluates major frontier agents for both security (attack resistance) and utility (legitimate compliance).
  - Reports high vulnerability rates, including harmful outcomes like exfiltration and destructive actions.
  - Concludes scaling and simple filtering are insufficient; calls for context-aware authorization.
  - Provides a concrete stress test for third-party skill ingestion pipelines.
- **Novelty:**
  - Benchmark specifically targeting skill/package-level injection vectors.
  - Joint utility-security metric framing for deployability.
- **Security relevance:**
  - **Permissions/authorization**: strengthens case for least-privilege skill sandboxing and trust tiers.
- **Limitations / open questions:**
  - Benchmark evolution pace vs rapidly changing agent frameworks.
  - Coverage for multi-skill collusion attacks.
- **Tags:** `skill-supply-chain`, `authorization`, `benchmark`, `prompt-injection`, `least-privilege`

## 6) OMNI-LEAK: Orchestrator Multi-Agent Network Induced Data Leakage

- **arXiv:** 2602.13477v2 (**cs.AI**)
- **Summary (5-8 lines):**

  - Finds single indirect injection can compromise multiple agents and trigger sensitive leakage.
  - Notably, leakage occurs even with baseline data access controls in place.

  - Emphasizes multi-agent threat models are not just scaled single-agent ones.

- **Novelty:**

- **Security relevance:**
  - **Data security + permissions**: agent-to-agent minimization and compartmentalized delegation become mandatory.
- **Limitations / open questions:**

## 10-line trend summary
1. The field is moving from single-turn jailbreak framing to **multi-turn control takeover** in tool-using agents.  
2. Indirect prompt injection remains the central practical threat, especially through tool outputs and retrieved artifacts.  
3. Data leakage work increasingly targets **silent channels** (egress paths, sharded leaks) that bypass output-only filters.  
4. Monitor/oversight-only strategies are being challenged; architectural controls are becoming the recommended baseline.  
5. Skill/plugin ecosystems are now treated as an explicit **agent supply-chain** security problem.  
6. Authorization and capability design is shifting toward trust tiers, least privilege, and context-aware enforcement.  
7. Memory research is maturing from storage convenience toward causal retrieval and lifecycle curation.  
8. Privacy work is expanding beyond PII regexes to contextual and semantic-sensitive leakage control.  
9. **Gap/opportunity #1:** standardized benchmarks for permissioned tool-use policies with formal guarantees are still missing.  
10. **Gap/opportunity #2:** memory-security benchmarks (poisoning, provenance tamper, stale-memory abuse) lag behind functional memory benchmarks.
