## 1) Real Money, Fake Models: Deceptive Model Claims in Shadow APIs
- **Authors:** Yage Zhang et al.
- **arXiv:** 2603.01919 (`cs.AI`, `cs.CR`, `cs.SE`)
- **Summary (5–8 lines):**
  - Audits whether shadow API providers actually serve the frontier models they claim.
  - Identifies 17 shadow APIs used by 187 papers, showing ecosystem-level exposure.
  - Runs multidimensional checks: utility gaps, safety behavior drift, and fingerprint-based identity tests.
  - Finds up to 47.21% performance divergence versus official APIs.
  - Reports safety unpredictability and 45.83% failure in model identity verification.
  - Concludes that shadow APIs can silently invalidate reproducibility and safety assumptions.
- **Novelty:**
  - First systematic side-by-side audit of official vs shadow LLM APIs at this scale.
  - Connects deception risk directly to scientific reproducibility and benchmark integrity.
- **Security relevance:**
  - **Permissions/data governance:** model provenance and API trust boundaries are foundational controls.
  - If model identity is spoofed, all downstream policy assumptions (safety filters, data handling) may fail.
- **Limitations / open questions:**
  - Coverage still limited to sampled shadow providers.
  - Need continuous monitoring since providers can change backend models quickly.
- **Tags:** shadow APIs, model provenance, reproducibility, safety drift, API security

## 2) Organizing, Orchestrating, and Benchmarking Agent Skills at Ecosystem Scale
- **Authors:** Hao Li et al.
- **arXiv:** 2603.02176 (`cs.CL`)
- **Summary (5–8 lines):**
  - Proposes AgentSkillOS for selecting and composing skills in very large skill ecosystems.
  - Introduces capability-tree organization for retrieval and DAG-based orchestration for execution.
  - Evaluates across 30 artifact-rich tasks and ecosystem scales up to 200K skills.
  - Shows structured retrieval approximates oracle selection better than flat invocation.
  - Demonstrates DAG orchestration materially improves task quality over naive skill calling.
  - Implies that agent capability governance needs structure, not just bigger tool catalogs.
- **Novelty:**
  - Ecosystem-scale benchmark for skill invocation quality across large skill inventories.
  - Explicit two-stage architecture (manage + solve) with measurable orchestration gains.
- **Security relevance:**
  - **Permissions/capability-policy:** capability trees can become policy surfaces for least-privilege tool routing.
  - Structured orchestration may reduce unsafe arbitrary tool invocation patterns.
- **Limitations / open questions:**
  - Security controls are implied, not directly red-teamed.
  - Need explicit deny/allow policy integration and adversarial skill tests.
- **Tags:** skill orchestration, capability routing, tool governance, agent systems, least privilege

## 4) VoiceAgentRAG: Solving the RAG Latency Bottleneck in Real-Time Voice Agents Using Dual-Agent Architectures
- **Authors:** Jielin Qiu et al.
- **arXiv:** 2603.02206 (`cs.SD`)
- **Summary (5–8 lines):**
  - Proposes dual-agent architecture: background “Slow Thinker” + foreground “Fast Talker.”
  - Slow agent predicts follow-up topics and prefetches chunks into semantic cache.
  - Fast agent answers from sub-millisecond cache for low-latency voice interaction.
  - Decouples retrieval timing from response generation, reducing real-time bottlenecks.
  - Positions semantic caching as a practical memory router for voice assistants.
- **Novelty:**
  - Predictive prefetch + dual-agent split for latency-critical conversational RAG.
  - Memory-router framing that operationalizes retrieval as continuous background process.
- **Security relevance:**
  - **Memory/data:** cache becomes a sensitive data plane requiring strict TTL, provenance, and access boundaries.
  - Prefetch logic can amplify over-collection risk without policy constraints.
- **Limitations / open questions:**
  - Limited discussion of cache poisoning or privacy leakage from predictive prefetch.
  - Needs policy-aware cache invalidation and audit trails.
- **Tags:** voice agents, semantic cache, RAG latency, memory routing, data minimization
