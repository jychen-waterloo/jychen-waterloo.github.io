### 1) Maris: A Formally Verifiable Privacy Policy Enforcement Paradigm for Multi-Agent Collaboration Systems
- **Authors:** Jian Cui, Zichuan Li, Luyi Xing, Xiaojing Liao  
- **arXiv:** `2505.04799v3` (**cs.CR**, replace announcement in window)
- **Summary (5-8 lines):**
  - Proposes **Maris**, a privacy-enhanced development paradigm for multi-agent collaboration systems.
  - Core mechanism is embedding **reference monitors** into key multi-agent conversation components.
  - Targets data leakage during inter-agent communication and external interaction with tools/users/LLMs.
  - Implemented in AutoGen and LangChain to demonstrate framework-level integration.
  - Introduces a dedicated privacy assessment framework with multiple threat scenarios.
  - Reports strong leakage mitigation while maintaining high task success.
- **Novelty:**
  - Formalizable message-flow control inserted directly into mainstream agent frameworks.
  - Privacy enforcement treated as a first-class architectural primitive, not post-hoc detection.
- **Security relevance:**
  - **Permissions/data-governance:** enforces what can flow where between agents and tools, reducing prompt-injection-driven exfiltration paths.
- **Limitations / open questions:**
  - Evaluation breadth across heterogeneous real production toolchains remains unclear.
  - Potential latency/complexity costs under large multi-agent graphs need deeper quantification.
- **Tags:** `multi-agent`, `privacy policy enforcement`, `reference monitor`, `prompt injection`, `data leakage`

### 2) AudAgent: Automated Auditing of Privacy Policy Compliance in AI Agents
- **Authors:** Ye Zheng, Yidan Hu  
- **arXiv:** `2511.07441v4` (**cs.CR, cs.AI**, replace-cross announcement in window)
- **Summary (5-8 lines):**
  - Presents an end-to-end runtime auditing stack to verify agent behavior against stated privacy policies.
  - Uses cross-LLM voting to formalize natural-language policies into machine-checkable representations.
  - Annotates runtime traces for sensitive-data handling using lightweight analyzers.
  - Applies ontology + automata checking for online compliance validation.
  - Provides UI-level traceability to show violations and interventions.
  - Empirically identifies practical policy gaps (e.g., SSN handling) and unsafe third-party tool interactions.
- **Novelty:**
  - Continuous, infrastructure-agnostic policy/runtime conformance loop for agents.
  - “Audit + block” operation rather than passive logging-only governance.
- **Security relevance:**
  - **Data security/privacy + audit:** directly addresses runtime policy drift and unauthorized sensitive-data processing.
- **Limitations / open questions:**
  - Policy formalization robustness under ambiguous legal language is still a brittle point.
  - Cross-domain transfer (non-English, sector-specific policies) not fully established.
- **Tags:** `privacy auditing`, `runtime monitoring`, `policy compliance`, `sensitive data`, `agent governance`

### 3) Learning When to Act or Refuse: Guarding Agentic Reasoning Models for Safe Multi-Step Tool Use
- **Authors:** Aradhye Agarwal, Gurdit Siyan, Yash Pandya, Joykirat Singh, Akshay Nambi, Ahmed Awadallah  
- **arXiv:** `2603.03205v1` (**cs.CL**, new)
- **Summary (5-8 lines):**
  - Introduces **MOSAIC**, a post-training framework for safer long-horizon tool-using agents.
  - Structures inference as a **plan → safety-check → act/refuse** loop.
  - Trains with preference-based RL on pairwise trajectories to capture safety distinctions missed by scalar rewards.
  - Evaluated zero-shot across multiple model families and OOD scenarios.
  - Benchmarks include harmful tasks, prompt injection, and privacy-leakage settings.
  - Reports reductions in harmful behavior and improved harmful-task refusal without hurting benign performance.
- **Novelty:**
  - Refusal modeled as an explicit first-class action in multi-step trajectories.
  - Preference-based safety optimization tailored to tool-augmented rollouts.
- **Security relevance:**
  - **Permission/policy + data leakage:** improves decision-time control over unsafe tool use and injection-triggered behavior.
- **Limitations / open questions:**
  - Reliance on quality/diversity of preference data.
  - Unknown robustness under adaptive adversarial tool feedback loops.
- **Tags:** `agent safety`, `tool use`, `refusal learning`, `prompt injection`, `post-training`

### 6) Zero-Permission Manipulation: Can We Trust Large Multimodal Model Powered GUI Agents?
- **Authors:** Yi Qian, Kunwei Qian, Xingbang He, Ligeng Chen, Jikang Zhang, Tiantai Zhang, Haiyang Wei, Linzhang Wang, Hao Wu, Bing Mao  
- **arXiv:** `2601.12349v2` (**cs.CR, cs.AI, cs.SE**, updated in window)
- **Summary (5-8 lines):**
  - Studies Android GUI agents under a broken “visual atomicity” assumption.
  - Introduces **Action Rebinding** where unprivileged malicious apps redirect agent actions during observe-act gaps.
  - Exploits task recovery and UI state preservation to construct multi-step attack chains.
  - Adds Intent Alignment Strategy to coerce agents through verification dialogs.
  - Evaluated across six agents and fifteen tasks with very high attack success.
  - Highlights architectural integration flaws between agents and OS UI semantics.
- **Novelty:**
  - Demonstrates high-impact attacks with **zero dangerous permissions** by exploiting timing/interaction semantics.
  - Connects UI race conditions with agent reasoning vulnerabilities.
- **Security relevance:**
  - **Authorization/capability abuse:** shows how effective privileges exceed declared app permissions in agent-mediated control paths.
- **Limitations / open questions:**
  - Mitigation design space (transactional UI snapshots, trusted execution lanes) still underexplored.
  - Generalization beyond tested mobile stacks needs broader evidence.
- **Tags:** `GUI agents`, `permission bypass`, `action rebinding`, `mobile security`, `agent-OS boundary`

## 10-line trend summary

1. The strongest thread is a shift from generic “safe prompting” to **enforceable control planes** (capabilities, policy automata, reference monitors).  
2. Agent security papers increasingly model failures as **authorization boundary errors**, not just model misalignment.  
3. Prompt-injection work is moving toward **covert channels** (Unicode/format-layer attacks) that bypass human review.  
4. Runtime auditing is becoming practical, with policy-to-execution traceability replacing static policy documents.  
5. Multi-agent communication is now treated as a first-class leakage surface requiring explicit flow control.  
6. Memory is emerging as both a capability multiplier and a persistent attack surface (poisoning, provenance drift, unsafe retrieval).  
7. Benchmarks are becoming more deployment-relevant (sandbox escape, tool-augmented harms, OOD injections).  
8. There is growing convergence between formal methods and agent engineering, especially around measurable guarantees.  
9. **Gap/opportunity #1:** unified evaluation that jointly measures capability safety, data governance, and memory poisoning resilience is still missing.  
10. **Gap/opportunity #2:** practical blueprints for low-latency, fail-closed defenses in high-throughput production agents remain underdeveloped.
