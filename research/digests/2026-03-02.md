## Paper list

### 1) OMNI-LEAK: Orchestrator Multi-Agent Network Induced Data Leakage
- **Authors:** Akshat Naik, Jay Culligan, Yarin Gal, Philip Torr, Rahaf Aljundi, Alasdair Paren, Adel Bibi  
- **arXiv:** 2602.13477v2 (**cs.AI**)
- **Summary (5–8 lines):**
  - Studies orchestrator-style multi-agent systems rather than single-agent setups.
  - Demonstrates a practical attack path where one indirect prompt injection propagates through delegated agents.
  - Leakage succeeds even when baseline data access control is present.
  - Evaluates susceptibility across reasoning and non-reasoning frontier models.
  - Shows attacker success without deep insider knowledge of implementation details.
  - Argues existing single-agent threat models understate multi-agent privacy risk.
- **Novelty:**
  - Concrete multi-agent leakage attack model with delegation-chain compromise.
  - Evidence that access-control-only designs fail under orchestration dynamics.
- **Security relevance:**
  - **Permissions + data security:** shows policy boundaries collapse via cross-agent delegation and context relay.
- **Limitations / open questions:**
  - Mostly one orchestration pattern; generalization to other topologies needs validation.
  - Defense cost/latency tradeoffs are not deeply quantified.
- **Tags:** multi-agent, prompt-injection, data-leakage, access-control, orchestration

### 2) Silent Egress: When Implicit Prompt Injection Makes LLM Agents Leak Without a Trace
- **Authors:** Qianlong Lan, Anuj Kaul, Shaun Jones, Stephanie Westrum  
- **arXiv:** 2602.22450v1 (**cs.CR**, **cs.AI**)
- **Summary (5–8 lines):**
  - Defines “implicit prompt injection” through URL previews/metadata/snippets.
  - Shows agents can be induced to exfiltrate sensitive runtime context via outbound network calls.
  - Reports high attack success in a local reproducible agent testbed.
  - Most successful attacks evade output-only safety checks.
  - Introduces “sharded exfiltration” to bypass simple DLP-like detection.
  - Finds network/system controls outperform prompt-only mitigations.
- **Novelty:**
  - Frames **network egress** itself as the primary security outcome for agent systems.
  - Quantifies stealth leakage via multi-request sharding.
- **Security relevance:**
  - **Data security/privacy:** direct exfiltration path triggered by indirect context poisoning.
- **Limitations / open questions:**
  - Validation across more model families/runtime stacks is needed.
  - Interaction with enterprise proxy/egress gateways remains underexplored.
- **Tags:** exfiltration, indirect-prompt-injection, egress-control, agent-security, DLP

### 3) Skill-Inject: Measuring Agent Vulnerability to Skill File Attacks
- **Authors:** David Schmotz, Luca Beurer-Kellner, Sahar Abdelnabi, Maksym Andriushchenko  
- **arXiv:** 2602.20156v3 (**cs.CR**, **cs.LG**)
- **Summary (5–8 lines):**
  - Examines the security risk introduced by third-party “skills” in LLM agents.
  - Builds a benchmark of 202 injection-task pairs spanning obvious and subtle attacks.
  - Measures both utility and safety under attack.
  - Finds high vulnerability rates, including execution of severe harmful behaviors.
  - Suggests scaling and simple filtering are insufficient mitigations.
  - Motivates context-aware authorization around skill/tool boundaries.
- **Novelty:**
  - First focused benchmark for skill-file injection in emerging agent supply chains.
  - Joint utility-security evaluation framework.
- **Security relevance:**
  - **Permissions/authorization:** highlights need for scoped trust and capability gating per skill.
- **Limitations / open questions:**
  - Benchmark coverage for closed-source commercial agent stacks is unclear.
  - Limited longitudinal evaluation of adaptive defenders.
- **Tags:** skill-supply-chain, authorization, prompt-injection, benchmarks, tool-security

### 4) AgentSentry: Mitigating Indirect Prompt Injection in LLM Agents via Temporal Causal Diagnostics and Context Purification
- **Authors:** Tian Zhang et al.  
- **arXiv:** 2602.22724v1 (**cs.CR**, **cs.AI**)
- **Summary (5–8 lines):**
  - Treats indirect prompt injection as temporal causal takeover across multi-turn trajectories.
  - Uses counterfactual re-execution at tool-return boundaries to localize compromise points.
  - Applies context purification to remove attack-induced deviations while preserving useful evidence.
  - Evaluated across AgentDojo suites and multiple attack families.
  - Reports strong utility under attack improvements over baselines.
  - Emphasizes continuation-safe defense vs blanket blocking.
- **Novelty:**
  - Causal diagnosis + purification pipeline at inference time.
  - Defense objective balances resilience and task completion.
- **Security relevance:**
  - **Data + permission safety:** reduces attacker steering of tool calls and downstream data actions.
- **Limitations / open questions:**
  - Runtime overhead from counterfactual replay may be substantial.
  - Robustness against adaptive attackers exploiting purification heuristics remains open.
- **Tags:** causal-defense, indirect-injection, tool-agents, inference-time-security, robustness

### 5) CourtGuard: A Model-Agnostic Framework for Zero-Shot Policy Adaptation in LLM Safety
- **Authors:** Umid Suleymanov et al.  
- **arXiv:** 2602.22557v1 (**cs.AI**, **cs.LG**)
- **Summary (5–8 lines):**
  - Recasts safety checking as retrieval-grounded evidentiary debate among agents.
  - Decouples policy logic from model weights to support rapid policy updates.
  - Achieves strong benchmark performance without fine-tuning.
  - Demonstrates zero-shot transfer by swapping policy corpora.
  - Also used for dataset curation and audit of adversarial examples.
  - Positions architecture as governance-friendly safety infrastructure.
- **Novelty:**
  - Practical zero-shot policy adaptation via externalized policy retrieval.
  - Safety system doubles as an auditing/data-curation workflow.
- **Security relevance:**
  - **Permissions/policy dimension:** stronger, more adaptable policy enforcement pipeline.
- **Limitations / open questions:**
  - Debate-agent reliability under collusion or shared blind spots is unresolved.
  - Retrieval quality becomes a critical failure point.
- **Tags:** policy-enforcement, governance, retrieval-augmented-safety, auditing, multi-agent

### 8) Towards Autonomous Memory Agents
- **Authors:** Xinle Wu, Rui Zhang, Mustafa Anis Hussain, Yao Lu  
- **arXiv:** 2602.22406v1 (**cs.AI**)
- **Summary (5–8 lines):**
  - Moves memory agents from passive recall to active knowledge acquisition and curation.
  - Introduces cost-aware extraction cascade across self/teacher/tools/expert channels.
  - Uses semantic-aware Thompson sampling for memory exploration/exploitation.
  - Reports gains on both verifiable and non-verifiable tasks.
  - Demonstrates improved adaptation without full model retraining.
  - Makes memory management a first-class control loop.
- **Novelty:**
  - Active memory acquisition with explicit cost/performance control.
  - Exploration policy tailored for memory cold-start and uncertainty.
- **Security relevance:**
  - **Memory dimension:** expands attack/poisoning surface at write-time and validation-time.
- **Limitations / open questions:**
  - Safety checks for poisoned external knowledge acquisition need stronger treatment.
  - Provenance/accountability for memory writes is under-specified.
- **Tags:** memory-agents, knowledge-curation, lifecycle, retrieval, adaptive-systems

## Top 3 must-read + reading order

2. **Silent Egress (2602.22450v1)** — Next, read for concrete exfiltration mechanics and why network-level controls must be first-class in agent security.

## 10-line trend summary
1. The center of gravity is shifting from single-agent prompt safety to **system-level agent security**.
2. Orchestrator and tool ecosystems are repeatedly shown to bypass nominal access controls.
3. Exfiltration is increasingly operationalized as **egress behavior**, not just harmful text output.
4. Inference-time defenses are moving toward causal/trajectory-aware mechanisms instead of static filters.
5. Policy logic is being externalized (retrieval + debate) to improve adaptability under changing governance rules.
6. Memory is evolving from passive storage to active acquisition/curation, increasing utility and risk simultaneously.
7. Formal methods (e.g., control barriers) are beginning to wrap LLM actions in verifiable safety envelopes.
8. Framework papers emphasize lifecycle governance and auditability, but empirical attacker-adaptive testing still lags.
9. **Gap/opportunity #1:** standardized benchmarks for permissioned multi-agent systems with explicit capability scopes and audit logs.
10. **Gap/opportunity #2:** end-to-end memory security frameworks (provenance, write-validation, revocation, poisoning recovery) for long-lived agents.
