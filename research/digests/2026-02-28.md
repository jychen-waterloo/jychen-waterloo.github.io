### 1) AgentSentry: Mitigating Indirect Prompt Injection in LLM Agents via Temporal Causal Diagnostics and Context Purification
- **Authors:** Tian Zhang et al.
- **arXiv ID:** 2602.22724 (cs.CR; cross-listed cs.AI)
- **Summary:**
  - Proposes an inference-time defense for indirect prompt injection (IPI) in tool-augmented agents.
  - Models IPI as **temporal causal takeover** rather than isolated toxic spans.
  - Uses controlled counterfactual re-execution at tool-return boundaries to localize takeover points.
  - Introduces context purification that removes attack-induced deviations while preserving task evidence.
  - Evaluated on AgentDojo across multiple task suites, attacks, and black-box LLM backends.
  - Reports elimination of successful attacks and large utility-under-attack gains over prior baselines.
- **Novelty:**
  - Causal takeover framing for multi-turn IPI.
  - Practical recovery path (continue task safely) instead of hard-stop blocking.
- **Security relevance:** Permissions/data — directly reduces adversarial control over tool execution trajectories and downstream data exposure.
- **Limitations / open questions:**
  - Added runtime cost from counterfactual replays at scale.
  - Robustness under adaptive attackers who anticipate purification.
- **Tags:** prompt-injection, tool-agents, causal-defense, runtime-guardrails, agentdojo

## 10-line trend summary
1. Runtime agent security is shifting from single-turn filtering to **trajectory-level control**.
2. Prompt injection research is converging on **tool-boundary and context-supply-chain attacks**.
3. A recurring finding: **safe text != safe actions**; action-channel auditing is becoming mandatory.
4. Governance is moving toward **contracts/policy artifacts** with enforceable runtime semantics.
5. Retrieval security is broadening beyond poisoning to **index-structure manipulation** (hubness effects).
6. Multi-tenant inference introduces serious **side-channel privacy** risks (KV-cache leakage economics improved).
7. Memory has become a first-class axis: benchmarking and training advances are accelerating.
8. But persistent memory expands attack persistence unless write/read lifecycle controls mature.
9. **Gap/opportunity #1:** Unified benchmark that jointly tests permissions, tool calls, memory poisoning, and data exfiltration in one long-horizon suite.
10. **Gap/opportunity #2:** Practical least-privilege frameworks for dynamic tool/skill ecosystems (capability scoping + provenance + revocation) remain underdeveloped.
